# Project Hardening Design

## 背景

本轮改进来自项目只读分析，目标是降低 bootstrap 后续维护成本，并修正文档、模板和运行时技能之间的细节偏差。

## 范围

本次只做低风险工程化调整：

- 修正文档示例与实际配置模板不一致的问题。
- 让 workspace 默认文件与文档建议一致。
- 调整平台列表里企业微信公网提示，避免把“视模式”误写成固定需要公网。
- 增加 CI 与 Release 构建工作流。
- 清理当前未使用的辅助函数。

## 不做

- 不重构 `main.go` 的大结构。
- 不新增非交互 bootstrap 模式。
- 不引入 TOML 解析库替换现有字符串回填逻辑。

这些改动影响面更大，适合单独设计和测试。

## 测试

- 代码行为改动先补 Go 测试。
- 文档、技能和 GitHub Actions 通过人工检查与 `go test ./...` 验证基本一致性。
