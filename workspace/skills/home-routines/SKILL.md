# 家庭日常事务

处理家庭里的提醒、待办、周期事项和设备相关请求。先读 `members.md`、`devices.md`、`tasks.md`，再决定是否需要调用 cc-connect 或外部工具。

## 适用场景

- 添加或查询家庭提醒、待办、周期任务
- 整理今天/本周家庭安排
- 账单、缴费、保修、采购等周期事项
- 设备控制前先确认风险和入口

## 执行步骤

1. 先判断这是临时问答还是会重复出现的流程。
2. 会重复出现的流程，优先更新 `tasks.md` 或相关 Markdown，而不是只给口头答复。
3. 需要定时提醒时，先确认时间、对象、是否主动发消息，再参考 `skills/cc-connect/SKILL.md` 创建 cron。
4. 涉及设备控制时，先查 `devices.md` 的风险等级和控制入口。
5. 没有明确设备入口或控制方式时，只给步骤建议，不直接执行。

## 安全边界

- 门锁、摄像头、门铃、支付、转账、删除文件、修改网络：必须二次确认。
- 不要保存或回传 API key、token、账号密码、完整配置文件。
- 不要把家庭成员敏感隐私写入 git 或公开文件。

## 失败处理

- 如果缺少成员称呼、时间或设备信息，先问一句补齐。
- 如果 cron 创建失败，说明失败原因，并给出可手动重试的命令。
- 如果只是普通查询，不要主动创建定时任务。
